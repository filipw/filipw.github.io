Lowering in the Compiler

A common technique (...) is to have the compiler “lower” from high-level language features to low-level language features in the same language.

- Eric Lippert

Lowering allows new C# features to be implemented (mostly) without changes in CLR/CoreCLR.

Lowering examples

  • 📕 loops
    IL only has branch which acts as goto
  • 📘 using blocks
    try/finally
  • 📙 lambdas
    translated into ordinary methods
  • 📗 iterators, async/await
    massive, complex state machines

The tuple case

    
var gift1 = ("chocolate", 5);
var gift2 = ("socks", 2);

Console.WriteLine(gift1 == gift2);

Lowered to...

    
Console.WriteLine(
    valueTuple.Item1 == valueTuple2.Item1 && 
    valueTuple.Item2 == valueTuple2.Item2
);

If someone wrote their own ValueTuple types (...) we would handle this case with tuple comparison instead of relying on the user-defined comparison.

- C# 7.3 tuple equality speclet
